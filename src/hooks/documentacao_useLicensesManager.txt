1. A Busca de Dados (Os Listeners)
========================================================================================================================================
  O useEffect cria dois "ouvintes" (listeners) em tempo real que trabalham juntos:
  Listener de Clientes: Ele busca a coleção customers para criar um "mapa" (customerMap). Isso serve para associar o ID do cliente ao nome real dele (ex: ID_123 -> "Posto Ipiranga"). Sem isso, sua tela só mostraria códigos.
  Listener de Unidades (Collection Group): Usamos o collectionGroup para buscar todas as subcoleções chamadas companies no banco todo de uma vez só.
  O "Pulo do Gato" do customerId: Como as unidades estão dentro de clientes, o código faz d.ref.parent.parent?.id. Isso "sobe" dois níveis no caminho do banco de dados para descobrir quem é o dono daquela unidade.


========================================================================================================================================
2. A Organização (O Agrupamento)
========================================================================================================================================
  Depois de receber a lista bruta de todas as empresas, o código usa a função .reduce() para transformar uma lista "bagunçada" em algo organizado por grupos:
  Ele olha o customerId de cada empresa.
  Se o grupo desse cliente ainda não existe no objeto groupedData, ele cria um novo bloco com o nome que pegamos no customerMap.
  Ele "empurra" a empresa para dentro da lista desse grupo.
  Resultado visual para o seu código:
  JSON
  {
    "ID_CLIENTE_A": { "customerName": "Grupo A", "companies": [...] },
    "ID_CLIENTE_B": { "customerName": "Grupo B", "companies": [...] }
  }


========================================================================================================================================
3. O Motor de Atualização (writeBatch)
========================================================================================================================================
  Essa é a parte que torna o sistema profissional. Em vez de fazer vários pedidos ao Firebase (o que seria lento e caro), usamos o Batch.
  updateBatch: É uma função genérica. Você entrega uma lista de empresas e o que quer mudar. Ela percorre cada empresa e prepara a alteração na memória.
  batch.commit(): É o comando final. Ele envia todas as alterações de uma vez só como uma única "bala". Ou muda tudo, ou não muda nada se der erro (atomicidade).
  Tratamento de Erros: O try/catch garante que, se o Firebase recusar a mudança (por falta de internet ou erro de permissão), o estado error receba uma mensagem e você possa avisar o usuário na tela.


========================================================================================================================================
4. Os Atalhos (Helpers)
========================================================================================================================================
  Para facilitar sua vida na hora de criar os botões na tela, o hook exporta três versões da mesma função:
  updateSingle: Você passa 1 empresa. (Ex: Botão na linha da empresa).
  updateGroup: Você passa a lista de empresas de um grupo específico. (Ex: Botão no cabeçalho da div do grupo).
  updateGlobal: Ele usa a lista allCompanies. (Ex: Botão no topo da página que muda tudo).

    Esse trecho é o "organizador de arquivos" do seu sistema. Ele pega uma lista bagunçada de todas as empresas e as separa em pastas baseadas em quem é o dono (o Cliente/Grupo).

    Aqui está o resumo do processo:

    O Ponto de Partida: Começamos com uma caixa vazia ({}).

    A Triagem: Para cada empresa na lista, verificamos se ela tem um "pai" (customerId). Se não tiver, ela é descartada do agrupamento para não causar erros.

    A Criação da Pasta: Se for a primeira vez que encontramos uma empresa daquele cliente específico, o código cria uma "pasta" nova. Nessa pasta, ele já coloca:

    O Nome Real do cliente (buscado naquele mapa que explicamos antes).

    Uma Lista Vazia pronta para receber as unidades.

    O Arquivamento: Com a pasta pronta, o código "joga" a empresa atual para dentro da lista de empresas daquele cliente.

    O Revezamento: O return acc garante que o trabalho feito até agora seja passado para a próxima empresa da lista, até que todas estejam guardadas em seus devidos lugares.
